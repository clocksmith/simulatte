# Simulatte Verification Ledger

Status: current implementation and remaining proof work. A checked item names an
executable contract and its verification command. It does not claim more than the
runtime measures.

## Numerical integrity

### Integrators `[done]`

Every solver module under
`public/blank/pipeline/phase-05-simulation/solvers/` exposes:

```json
{
  "integrator": {
    "scheme": "explicit_euler_v1 | semi_implicit_euler_v1",
    "order": 1,
    "symplectic": false,
    "stableDt": 0.05,
    "cfl": 0.9,
    "stateContract": ["channel-family"]
  }
}
```

The registry validates metadata for all 23 operator mappings. The compiler copies
the selected contract into every solver step. A requested frame delta larger than
the smallest stable delta is divided into deterministic global substeps; elapsed
simulation time is never silently discarded.

Rotational mechanics uses the implemented semi-implicit update. Other current
solvers use their implemented explicit update. Simulatte does not claim Verlet or
RK4: those schemes require derivative/state contracts the current solver modules do
not expose.

### Energy accounting `[done, fail-closed]`

Every solver state carries `simulatte.energyLedger.v1` with per-domain kinetic,
potential, thermal, and field components, step/cumulative discrepancy, source,
sink, dissipation, and tolerance band.

An energy band is only `accept`, `warn`, or `fail` when the solver graph declares
`energyAccounting.complete: true` and supplies its energy-flow contract. Otherwise
the band is `unmeasured`. A changing channel is not relabeled as input or loss just
to close the ledger.

### Checkpoint / snapshot / restore `[done]`

`simulatte.checkpoint.v1` captures the spec hash, frame, simulation time, seed,
integrator carry, typed-array channels, readouts, and content hash. Restore refuses
tampered content and cross-spec checkpoints. Float and integer typed arrays survive
serialize/deserialize/restore without becoming plain objects.

### Spatial-grid boundaries `[done]`

The solver compiler implements reflective, absorbing, periodic, and coupled
boundaries. Coupled faces use equal-and-opposite upwind transfer and conservative
overlap weights when neighboring grids have different resolutions. Boundary steps
fail closed when `|u| dt / dx` exceeds the declared CFL limit. Ghost layers support
the same boundary vocabulary.

### Determinism `[done]`

`npm run scan:determinism` scans 24 compute-path modules and rejects wall-clock,
entropy, timer, and animation APIs on the deterministic surface. Spec and remix IDs
are content-derived. Equal inputs produce equal IDs and checkpoint hashes.

Verification:

```sh
node --test --test-name-pattern='solver integrator|solver stepping|energy discrepancy|solver checkpoints|grid boundaries' tests/physical-compiler.test.cjs
npm run scan:determinism
```

## Remaining proof boundary

- Bind human recognizability verdicts to the six current gold screenshots. Machine
  packet/pixel obligations are necessary evidence, but they cannot self-certify
  that a cat, airplane, excavator, violin, or octopus is recognizable to a person.
- Keep the gold prompts sealed from catalog rows. Add reusable part, relation, pose,
  and material cards only when a failed obligation identifies a missing substrate.
- Re-run the live WebGPU/model audit after any retrieval, composition, card-index,
  packet, shader, or model-lock change. Static packet fields are not pixel proof.

The repository gate is complete only when unit/static checks, universe validation,
model-backed inference, live WebGPU screenshot proof, and hash-bound human
adjudication agree.
